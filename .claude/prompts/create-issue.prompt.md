# create-issue — Issue 作成

GitHub Issue を正しい粒度・構造で登録する。

## Issue 種別

| 種別     | いつ使うか                              |
| -------- | --------------------------------------- |
| **Task** | 実装タスク単位（1〜3 日で完了する粒度） |
| **Bug**  | コード上の不具合を発見したとき          |

## 登録前に必ず提示してレビューを受ける

1. 作成予定タイトル
1. 付与予定ラベル・マイルストーン
1. Issue 本文ドラフト（チェックリスト・完了条件を含む）

承認後に `gh issue create` を実行する。

## Task Issue テンプレート

```markdown
## 概要
<実装内容の概要>

**WBS ID**: <ID> | **依存**: <依存WBS> | **優先度**: 高/低

## 完了条件
<具体的な動作確認方法>

## 作業チェックリスト
- [ ] タスク1
- [ ] タスク2

## ビジネスルール（該当時）
- <制約・バリデーション仕様>
```

## Bug Issue テンプレート

```markdown
## 概要
<何が問題か・影響範囲>

## 再現手順
1. ...

## 期待する動作
...

## 問題箇所
`<ファイルパス>` L<行番号>
```

## gh コマンド

```bash
gh issue create \
  --title "[WBS-XXX] <タイトル>" \
  --label "<label>" \
  --milestone "<milestone>" \
  --body "..."
```
